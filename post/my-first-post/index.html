本文主要介绍利用hugo，github pages，和自己的域名，进行搭建简易静态博客的流程。

Hugo and Github pages

hugo 简单理解为一个命令行工具，能够根据Markdown文本格式生成静态的网页， github pages 可以根据你的内容（html，css等）展示网页效果。

步骤如下：

  • 申请一个github pages

    • 创建一个github 仓库，名字格式必须是github_user_name.github.io

  • 安装hugo， 生成一个静态博客，添加博客文档（md格式）

    • linux下，我本人是Ubuntu的系统，到 Hugo Release 页面下载deb，使用dpkg -i *******.deb进行安装
    • 安装后即可使用hugo 命令hugo version
      1. hugo new site blog_name
      2. cd blog_name
      3. 以 Even 主题为例子，各个主题略有不同.
        • git clone https://github.com/olOwOlo/hugo-theme-even themes/even
        • cp ./themes/even/exampleSite/config.toml ./
        • 编辑config.toml即可对主题进行配置
      4. hugo new post/some-content.md会生成content/post/some...md文件，编辑md文件即可，注意头部格式，需要将draft设置未false才能在正式部署到github pages后显示
      5. 验证方式为hugo server -D

  • 提交到GitHub pages

    • 生成github pages 需要的静态文件，在根目录执行hugo命令，生成一个public文件夹cd public
    • git init
    • git remote add origin https://github.com/your_name.your_name.github.io.git
    • git add .
    • git commit -m "first commit"
    • git push -u origin master
    • 然后访问https://your_name.github.io即可

绑定自己的域名

  • 域名商绑定到 your_name.github.io
  • 本地在public下面创建名字叫做CNAME的文件，文件内容为你域名如kkk.com，需要提交github 仓库里面。

虚拟机等本地局域网测试访问问题

  • 虚拟机做NAT端口转化， 将虚拟机ip的1313端口转化到本地的23233端口
  • hugo server 指定绑定的ip， hugo server --bind "0.0.0.0" -p 1313 -D

github 要求输入用户名密码问题

设置gitconfig.credential.helper=store
